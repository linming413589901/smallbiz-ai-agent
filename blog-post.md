# 我用 AI Agent 给小商家做了个智能客服，从零到上线只花了一周

## 为什么做这个？

去年帮一个开服装店的朋友处理线上客服，发现一个痛点：小商家每天要回答大量重复问题——"发货要几天？""能退换吗？""有没有货？"。雇个客服成本高，用自动回复又太死板。

大厂的客服机器人？太贵，而且不够灵活。

所以我想：能不能用 AI Agent 做一个"懂业务"的智能客服？它不只是关键词匹配，而是能真正理解顾客意图，调用工具查库存、查订单、回答常见问题，像真人一样聊天。

这就是 smallbiz-ai-agent 的起点。

## 技术选型

### 前端：Next.js

选 Next.js 的原因很简单：
- API Routes 可以直接写后端逻辑，不用单独部署服务
- Vercel 一键部署，零运维
- React 生态成熟，改 UI 快

### AI 框架：LangChain

LangChain 的核心价值是 **Tool Calling**（工具调用）。不是让模型瞎猜库存，而是：
1. 顾客问"有没有T恤" → 模型决定调用 `check_inventory` 工具
2. 工具返回真实数据 → 模型用自然语言组织回答
3. 顾客问"我的快递到哪了" → 模型调用 `query_order` 工具

这种"模型做决策 + 工具做执行"的架构，比纯聊天靠谱得多。

### 模型：MiMo (小米)

用的是小米的 MiMo-v2.5-pro 模型，通过 OpenAI 兼容 API 调用。选它的原因：
- 中文理解能力强（客服场景必须）
- 支持 Function Calling（LangChain 工具调用的前提）
- 性价比高，适合中小商家的预算

## 架构设计

整体架构很简洁：

```
用户输入 → Next.js API Route → LangChain Agent
                                    ↓
                              模型决策是否调用工具
                                    ↓
                    ┌───────────────┼───────────────┐
                    ↓               ↓               ↓
              faq_lookup      check_inventory    query_order
              (常见问题)        (库存查询)        (订单查询)
                    ↓               ↓               ↓
                    └───────────────┼───────────────┘
                                    ↓
                              模型组织语言回复
                                    ↓
                              返回给用户
```

三个工具覆盖了小商家客服的三大核心场景：

1. **FAQ 工具** — 营业时间、退换货、发货、包邮等常见问题
2. **库存工具** — 按商品名或类别查询库存和价格
3. **订单工具** — 按订单号或客户名查询状态和物流

## 遇到的坑

### 坑1：FAQ 关键词匹配太死板

最初 FAQ 工具用简单的 `includes` 匹配，顾客说"退货"能命中，但说"不要了""想退掉"就匹配不上。

解决：建了一个同义词别名映射表。"退货""退款""不想买了"都映射到"退换货"这个 FAQ 条目。命中率从 60% 提升到 90%+。

### 坑2：API Key 暴露

第一次开发时把 API Key 直接写在代码里，推到了 GitHub。虽然只是示例 Key，但这是个坏习惯。

解决：`.env.local` 存本地配置，`.gitignore` 排除，Vercel 用环境变量注入。

### 坑3：工具调用死循环

早期代码没有限制工具调用轮数，理论上模型可能无限循环调用工具。

解决：加了 `maxRounds = 5` 的硬限制，超过就强制返回。

### 坑4：Vercel 部署缺环境变量

本地跑得好好的，部署到 Vercel 直接报错"API key not found"。因为 `.env.local` 不会被推到线上。

解决：用 `vercel env add` 把所有环境变量手动加到 Vercel 项目里。
